# 官方初始化提示词

## 适用时机

- 新项目还没有 OpenSpec。
- 老项目准备按官方 OpenSpec 方式补装。

## 提示词

```text
请使用官方 OpenSpec 方式初始化当前项目，并保持官方目录结构不变。

执行要求：
1. 先运行 `openspec init --tools codex`
2. 保留官方生成的 `openspec/config.yaml`、`openspec/changes/`、`openspec/specs/`、`openspec/changes/archive/`、以及 `.codex` 下的 OpenSpec 技能
3. 不要自造新的主流程目录去替代 `openspec/`
4. 初始化完成后，检查当前项目的技术栈、主入口、目录职责、测试方式、风险边界
5. 把这些项目上下文补充到 `openspec/config.yaml`
6. 补根 `AGENTS.md`，并为每个包级、领域级目录补自己的 `AGENTS.md`
7. 如果项目需要更强治理，再补 `openspec/CODEX_MANUAL.md`
8. 输出最终新增或修改的文件，以及后续推荐怎么进入 `explore / propose / apply / review / archive`

额外要求：
- 保持官方 OpenSpec 的方式，不要把项目自定义规则伪装成官方命令
- 如果项目需要 `review`，把它当作 apply 后自动衔接的项目补充阶段说明
- `review` 完成后要停止并等待用户确认，不要默认直接 archive
- 不要只生成根 `AGENTS.md`，每个关键包目录都要有自己的 `AGENTS.md`
- 默认用中文说明，但保留 OpenSpec 结构关键字
```
