# 生成包级 AGENTS 提示词

## 适用时机

- 根 `AGENTS.md` 已经存在。
- 现在要把规则继续下沉到每个包级、业务域、模块级目录。

## 提示词

```text
请基于当前项目目录结构，为每个关键包级、领域级、模块级目录生成自己的 `AGENTS.md`。

执行要求：
1. 先梳理项目中的关键目录边界，不要只看根目录
2. 为每个关键目录生成一份局部 `AGENTS.md`
3. 根 `AGENTS.md` 只保留全局规则，不重复包内细节
4. 包级 `AGENTS.md` 至少覆盖：
   - 当前目录职责
   - 对外能力边界
   - 优先复用入口
   - 本目录特有的命名、测试、异常处理要求
   - 哪些规则比根目录更严格
5. 如果某些目录只适合继承全局规则，也请明确说明是否仍需要最小化 `AGENTS.md`

额外要求：
- 保持规则长期有效，不写一次性任务背景
- 涉及 OpenSpec 流程时，继续保持 `apply -> 自动 review -> 等待确认 -> archive`
- 输出时列出生成了哪些包级 `AGENTS.md`，以及各自负责的作用域
```
