# 探索需求提示词

## 适用时机

- 需求还在分析中。
- 还不确定 change 范围、方案或命名。

## 提示词

```text
请按官方 OpenSpec 的 explore 思路，先帮助我探索这个需求，不要直接进入实现。

请做这些事：
1. 先检查当前项目是否已有相关 active change 或现有 spec
2. 结合代码现状，梳理问题背景、影响范围、候选方案和主要风险
3. 如果有必要，用图表或清单帮助我比较方案
4. 如果结论还不稳，不要急着创建 proposal
5. 如果结论已经足够稳定，请给出推荐方案和建议的 change 名称

如果项目启用了严格治理，请在最后停在“第一确认闸门”：
- 汇总当前理解
- 汇总变更范围
- 汇总候选方案 / 推荐方案
- 给出建议的 change 名称

在我确认前，不要创建或修改 `openspec/changes/**` 文档。
```
